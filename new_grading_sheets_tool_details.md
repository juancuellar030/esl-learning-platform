As a teacher, I normally use Excel to do my grade sheets for the different students’ grades. From my experience using the Student Tracker tool, I found out that it is possible and viable to create a tool that mimics what I do in the Excel gradesheets. So use @brainstoming to redact an implementation plan for this new grade sheet tool. Let’s use the 200+ students and groups registered in this platform (the ones that can be imported in the Student Material Tracker tool – 3A, 3B, 3C, 4A, 4B… etc). Here’s a screenshot of what the layout looks like in the Excel version, so that we can build a tool that resembles this layout:
-	A library or dashboard of the saved grade sheets.
-	An intuitive way to set up or create a new sheet that contains all the necessary information (details per sheet in the following bullet points).
-	For the sheet itself, a header section that details the teacher’s name, the subject, the group or course, and the school term.
-	The main grid that contains the columns present in the screenshot (student’s number, name, grades columns and the last column that contains the descriptions of the activities to be graded during the term. This last ‘activities column’ should be divided into 3 categories (Competencia cognitiva, comp. laboral y comp. ciudadana and this column can be placed below the student’s grades columns for those devices with tighter screens)
-	For each activity description, there should be a corresponding grades column, so let’s make sure to add the ability to easily add or remove columns for each activity added.
-	A good export to CSV since the school normally accepts Excel files for the grading final report. Make sure the exported CSV files per sheet contains the everything (if possible, make the exported CSV layout closely resemble the one in the screenshot). Don’t forget to add support for tildes since the students’ names are mostly in Spanish.
-	Add a list to check/uncheck which allows me to control which created sheets to export.
-	This new tool should be closely linked to the Grading Scale tool, since in this tool I can set up the scales by which the calculated grades will be added to the grade sheets. So let’s add a way to migrate the generated grades to the grade sheets tool, or instead, let’s add a way to set up the grade scale system in this new grading tool. I’ll leave the integration of these tools to what you consider offer the best workflow between the two tools.
-	For new things to add that are not present or offered by Excel:
1. the color-coded grades present in this tool, depending on the minimum passing grade established.
2. A prediction or indicator if the student Is likely to fail the subject (keep in mind that each category (the 3 competencias) has a certain percentage weight: The Cognitiva and Laboral grades weight 35% and the Cuidadanas weight 30% to complete the total 100%, so use these percentages to calculate the prediction.
3. Highlight a student’s entire row in a sheet when clicked on the name of the student.
-	A way to drag around (move position), eliminate or temporarily hide any of the set up columns. For the moving columns feature, you can use the same system used for moving the columns in the student material tracker tool.
-	A visual indicator for when an entire sheet is fully filled out with all the grades and a system that notifies or informs me of the students that are missing grades and detail the activity description of the missing grades. For this report system, and to avoid unnecessary report of massive amounts of students per group, let’s start reporting when at least half of the students’ score in a specific group have been filled in.
-	Let’s make everything juicy and beautifully animated, and make sure it matches perfectly in terms of appearance and fonts to the overall platform’s look, along with a proper dark mode version. 
To avoid errors in the code, you can implement the creation of this new tool by phases.
